# 3. tcp 三次握手/ 四次挥手过程
#### 三次握手
1. 客户端发起 ，服务端接收
2. 服务端发出收到请求，客户端收到
3. 客户端发出对此次收到的收到信息，
问题 如果 客户端最后一次发出 ，服务端没有收到的结果？ 三次握手还能成功吗

#### 四次挥手
1. 客户端发出断开申请，服务端接收
2. 服务端发出对客户端请求的接收到请求，
3. 客户端收到 服务端发出的确认请求
4. 服务端断开？？
